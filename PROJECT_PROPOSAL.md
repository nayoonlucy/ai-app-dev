## 프로젝트 계획 제안서

## 1. 프로젝트 개요

### 1.1 프로젝트명
**GitHub Portfolio Uploader** (GitHub 포트폴리오 자동 업로드 앱)

### 1.2 프로젝트 목적
초급 개발자와 GitHub 사용이 어려운 사용자들을 위한 웹 기반 GitHub 파일 관리 도구 개발

### 1.3 개발 기간
**7일** (MVP - Minimum Viable Product 기준)

### 1.4 개발 방식
- MVP 방식 (최소 기능 구현)
- 반응형 웹 애플리케이션
- GitHub REST API 활용

## 2. 개발 배경 및 동기

### 2.1 문제 인식

현재 많은 초급 개발자와 학부생들이 GitHub를 활용하여 포트폴리오를 관리하고 프로젝트를 공유하려고 하지만, 다음과 같은 어려움에 직면하고 있습니다:

#### 2.1.1 기술적 진입 장벽
- **Git 명령어의 복잡성**: `git add`, `git commit`, `git push` 등 다양한 명령어를 기억하고 사용해야 함
- **터미널 사용의 어려움**: 명령줄 인터페이스에 대한 두려움과 불편함
- **환경 설정의 복잡성**: Git 초기 설정, SSH 키 관리, 원격 저장소 연결 등

#### 2.1.2 접근성 문제
- **컴퓨터 의존성**: Git은 주로 데스크톱 환경에서 사용되며, 모바일에서는 접근이 어려움
- **실시간 업로드 불가**: 컴퓨터가 없는 상황에서 프로젝트 파일을 즉시 업로드하기 어려움

#### 2.1.3 효율성 문제
- **반복 작업의 번거로움**: 매번 동일한 명령어를 입력해야 하는 번거로움
- **파일 관리의 비효율**: 여러 파일을 개별적으로 업로드해야 하는 불편함

### 2.2 해결 방안

위 문제들을 해결하기 위해 **웹 기반 GUI 애플리케이션**을 개발하여:
- ✅ 명령어 없이 마우스 클릭만으로 파일 업로드
- ✅ 드래그 앤 드롭으로 직관적인 파일 관리
- ✅ 모바일에서도 접근 가능한 반응형 디자인
- ✅ GitHub API를 활용한 안정적인 파일 업로드

## 3. 타겟 사용자

### 3.1 주요 타겟
1. **학부생**
   - 포트폴리오 프로젝트를 GitHub에 업로드하고 싶은 학생
   - Git 사용법을 아직 완전히 익히지 못한 학생

2. **초급 개발자**
   - GitHub 사용의 진입 장벽에 부딪힌 개발자
   - 명령어를 자주 잊어버리는 개발자

3. **GitHub 편의 사용자**
   - GitHub를 더 편리하게 사용하고 싶은 개발자
   - 빠르고 간단한 파일 업로드를 원하는 사용자

### 3.2 사용자 니즈
- 명령어 사용 없이 파일 업로드
- 컴퓨터가 없는 상황에서도 접근 가능
- 직관적이고 쉬운 사용자 인터페이스
- 빠르고 효율적인 파일 관리

---

## 4. 기능 명세

### 4.1 MVP (Minimum Viable Product) 기능

#### 4.1.1 인증 기능
- **GitHub Personal Access Token 기반 인증**
  - 사용자 토큰 입력을 통한 로그인
  - 토큰 유효성 검증
  - 로그인 상태 유지 (로컬 스토리지)

#### 4.1.2 레포지토리 관리
- **레포지토리 목록 조회**
  - 사용자의 모든 레포지토리 자동 로드
  - 레포지토리 정보 표시 (이름, 설명, 공개/비공개)
  - 레포지토리 선택 기능

- **새 레포지토리 생성**
  - 레포지토리 이름 입력
  - 설명 입력 (선택)
  - 공개/비공개 설정
  - 즉시 생성 및 선택

#### 4.1.3 파일 업로드
- **파일 선택**
  - 드래그 앤 드롭 지원
  - 파일 선택 다이얼로그
  - 다중 파일 선택 가능

- **파일 업로드**
  - 선택한 파일을 GitHub에 자동 커밋
  - 업로드 진행 상태 표시
  - 성공/실패 메시지 표시
  - GitHub 레포지토리 링크 제공

### 4.2 UI/UX 기능
- **반응형 디자인**: 모바일, 태블릿, 데스크톱 지원
- **직관적인 인터페이스**: 명확한 버튼과 안내 메시지
- **실시간 피드백**: 업로드 상태 및 결과 즉시 표시

## 5. 기술 스택

### 5.1 프론트엔드
- **React 18**: 사용자 인터페이스 구축
- **TypeScript**: 타입 안정성 및 코드 품질 향상
- **Vite**: 빠른 개발 서버 및 빌드 도구
- **Tailwind CSS**: 빠른 스타일링 및 반응형 디자인
- **React Router**: 페이지 라우팅

### 5.2 API 및 라이브러리
- **GitHub REST API**: 레포지토리 및 파일 관리
- **@octokit/rest**: GitHub API 클라이언트 라이브러리
- **lucide-react**: 아이콘 라이브러리

### 5.3 기술 스택 선택 이유
- **React + TypeScript**: 빠른 개발 속도와 타입 안정성
- **Vite**: 빠른 개발 서버 및 HMR (Hot Module Replacement)
- **Tailwind CSS**: 빠른 스타일링, 반응형 디자인 용이
- **GitHub REST API**: 안정적이고 문서화가 잘 된 API

## 6. 시스템 아키텍처

### 6.1 전체 구조
```
사용자 브라우저
    ↓
React 웹 애플리케이션
    ↓
GitHub REST API
    ↓
GitHub 레포지토리
```

### 6.2 주요 컴포넌트
- **AuthContext**: 인증 상태 관리
- **Login**: 로그인 화면
- **Dashboard**: 메인 대시보드
- **RepositorySelector**: 레포지토리 선택/생성
- **FileUploader**: 파일 업로드 컴포넌트

### 6.3 데이터 흐름
1. 사용자 토큰 입력 → 인증 확인
2. 레포지토리 목록 조회 → 화면 표시
3. 레포지토리 선택/생성 → 선택된 레포지토리 저장
4. 파일 선택 → 파일 정보 저장
5. 업로드 요청 → GitHub API 호출 → 결과 표시

## 10. 제약사항 및 고려사항

### 10.1 기술적 제약
- **GitHub API 제한**: 시간당 요청 제한 (5,000 requests/hour)
- **파일 크기 제한**: GitHub API는 100MB까지 지원 (더 큰 파일은 Git LFS 필요)
- **보안**: Personal Access Token을 프론트엔드에서 관리 (프로덕션에서는 백엔드 필요)

### 10.2 사용자 제약
- **GitHub 계정 필요**: 사용자는 GitHub 계정이 있어야 함
- **인터넷 연결 필요**: 온라인 환경에서만 사용 가능
- **토큰 생성 필요**: 사용자가 직접 Personal Access Token을 생성해야 함

### 10.3 해결 방안
- API 제한: 요청 최적화 및 캐싱 활용
- 보안: MVP 이후 백엔드 서버 구축으로 토큰 안전하게 관리
- 사용자 가이드: 토큰 생성 방법 상세 안내 제공
