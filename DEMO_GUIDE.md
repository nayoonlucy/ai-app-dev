# 시연 가이드 (Demo Guide)

## 시연 준비

### 1. 사전 준비
1. GitHub Personal Access Token 미리 생성 (repo 권한)
2. 테스트용 레포지토리 1-2개 준비 (또는 새로 만들 예정)
3. 업로드할 샘플 파일 준비 (예: portfolio.html, resume.pdf 등)

### 2. 앱 실행
```bash
# 의존성 설치 (처음 한 번만)
npm install

# 개발 서버 실행
npm run dev
```

브라우저에서 `http://localhost:3000` 자동으로 열림

## 시연 시나리오 (약 5-7분)

### 1단계: 로그인 화면 소개 (30초)
- **화면 설명**: "GitHub 포트폴리오 업로더는 초급 개발자와 GitHub 사용이 어려운 분들을 위한 앱입니다"
- **문제점 설명**: "명령어를 몰라도, 복잡한 Git 설정 없이도 간단하게 파일을 업로드할 수 있습니다"
- **로그인 방법**: Personal Access Token 입력 (GitHub에서 토큰 생성 링크 제공)

### 2단계: 레포지토리 관리 (1-2분)
- **기존 레포지토리 조회**: "제 GitHub 계정의 모든 레포지토리가 자동으로 불러와집니다"
- **새 레포지토리 생성**: 
  - "새로 만들기" 버튼 클릭
  - 이름: "my-portfolio"
  - 설명: "포트폴리오 프로젝트"
  - 공개/비공개 선택
  - 생성 완료 확인

### 3단계: 파일 업로드 (2-3분)
- **드래그 앤 드롭**: 
  - "파일을 드래그 앤 드롭하면 됩니다" (실제로 파일 드래그)
  - 여러 파일 동시 선택 가능
- **파일 선택**: 
  - "또는 클릭해서 파일을 선택할 수도 있습니다"
- **업로드 실행**:
  - "업로드하기" 버튼 클릭
  - 진행 상태 표시
  - 성공 메시지 확인
- **GitHub 확인**: 
  - "실제로 GitHub에 올라갔는지 확인해보겠습니다" (GitHub 링크 열기)

### 4단계: 모바일 반응형 (1분)
- **화면 크기 조절**: 
  - "모바일에서도 사용할 수 있도록 반응형으로 제작했습니다"
  - 브라우저 개발자 도구로 모바일 뷰 전환 (F12 > 디바이스 모드)

### 5단계: 핵심 가치 강조 (1분)
- **타겟 사용자**: "학부생, 초급 개발자, GitHub 진입 장벽에 부딪힌 분들"
- **해결한 문제**: 
  - ✅ 명령어 없이 사용 가능
  - ✅ 컴퓨터 없어도 모바일에서 접근 가능
  - ✅ 효율적인 파일 관리
- **MVP 기능**: 레포지토리 조회, 생성, 파일 업로드

## 시연 팁

### ✅ DO (해야 할 것)
- 실제로 파일을 업로드해서 GitHub에 올라가는 모습을 보여주기
- 에러가 나도 당황하지 말고 "이런 경우는..."이라고 설명
- 타겟 사용자의 입장에서 설명하기 ("명령어를 몰라도...")

### ❌ DON'T (하지 말아야 할 것)
- 너무 빠르게 진행하지 않기
- 기술적인 세부사항에 너무 깊이 들어가지 않기
- 코드를 보여주지 않기 (시연은 기능 중심)

## 대비 질문 및 답변

**Q: 보안은 괜찮나요?**
A: Personal Access Token은 브라우저 로컬 스토리지에만 저장되며, GitHub API 호출에만 사용됩니다. 실제 프로덕션에서는 OAuth 서버를 통해 더 안전하게 처리할 수 있습니다.

**Q: 큰 파일도 업로드 가능한가요?**
A: GitHub API 제한에 따라 파일당 100MB까지 가능합니다. 더 큰 파일은 Git LFS를 사용해야 합니다.

**Q: 폴더 구조는 어떻게 올리나요?**
A: 현재 MVP는 파일 단위 업로드를 지원합니다. 폴더 구조 업로드는 향후 개선 예정입니다.

**Q: 다른 사람의 레포지토리에도 올릴 수 있나요?**
A: 현재는 본인의 레포지토리에만 업로드 가능합니다. 협업 기능은 향후 추가 예정입니다.
